# CRM-система

Упрощённая CRM для учёта продавцов и их транзакций. Тестовое задание.

## Что внутри

- CRUD по продавцам и транзакциям
- Аналитика: самый продуктивный продавец за день/месяц/квартал/год, фильтр продавцов по сумме за период, поиск лучшего периода активности
- Soft delete для продавцов — история транзакций сохраняется
- Централизованная обработка ошибок

## Стек

Java + Spring Boot (Web, Data JPA, Validation), PostgreSQL, Gradle. Тесты — JUnit 5 + Mockito + MockMvc.

## Запуск

Нужна запущенная PostgreSQL.

```
url: jdbc:postgresql://localhost:5432/crm_db
username: postgres
password: postgres
```

Создать базу:

```sql
CREATE DATABASE crm_db;
```

Запустить приложение:

```bash
./gradlew bootRun
```

Поднимется на `http://localhost:8080`.

Креды можно переопределить через переменные окружения: `DB_USER`, `DB_PASSWORD`.

## Тесты

```bash
./gradlew test
```

## API

### Продавцы

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/sellers` | Список всех продавцов |
| `GET` | `/api/sellers/{id}` | Один продавец |
| `POST` | `/api/sellers` | Создать продавца |
| `PUT` | `/api/sellers/{id}` | Обновить продавца |
| `DELETE` | `/api/sellers/{id}` | Soft delete |


### Транзакции

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/transactions` | Все транзакции (опционально `?sellerId=N`) |
| `GET` | `/api/transactions/{id}` | Одна транзакция |
| `POST` | `/api/transactions` | Создать транзакцию |

Транзакции не редактируются и не удаляются


`paymentType`: `CASH`, `CARD`, `TRANSFER`. Если `transactionDate` не передан, проставится текущее время.

### Аналитика

**Самый продуктивный продавец за период.** Возвращает список — если у нескольких одинаковая максимальная сумма, вернутся все.

```
GET /api/analytics/most-productive?period=DAY
```

`period`: `DAY`, `MONTH`, `QUARTER`, `YEAR`.

**Продавцы с суммой меньше порога за период.**

```
GET /api/analytics/below-threshold?from=2026-05-01T00:00:00&to=2026-05-31T23:59:59&threshold=10000
```

В ответе будут и те продавцы, у которых вообще нет транзакций за период (их сумма = 0).

**Лучший период активности продавца.** Окно фиксированной длины, в которое
попало максимальное число транзакций. Длину задаёт клиент через `periodDays`,
даты начала и конца — результат алгоритма. При равенстве числа транзакций
выбирается окно с большей суммой.
```
GET /api/analytics/best-period?sellerId=1&periodDays=7
```

## Обработка ошибок

Все ошибки возвращаются единым форматом:

```json
{
  "timestamp": "2026-05-24T10:00:00",
  "status": 404,
  "error": "Not Found",
  "message": "Продавец с id=99 не найден",
  "path": "/api/sellers/99"
}
```

Для ошибок валидации добавляется `fieldErrors` с детализацией по полям.
